# Documentation Archive

One-off documents, test files, and superseded versions organized by date.

## Structure

```
_archive/
├── YYYY-MM-DD/
│   ├── tools_backup/    # Superseded tool versions
│   └── one_offs/        # Test files, status docs, implementation details
```

## Policy

- All test outputs, status reports, and implementation details go here automatically
- Keep root docs directories clean and focused on active documentation
- Archive superseded versions rather than deleting (enables rollback)
- Review quarterly and purge files older than 1 year if not needed
