Why not TACC alone?

The Core Vision

Digital Twins, Coming Online

Five digital twins across four facilities. The simulation pipelines exist. The sensor integration is in progress. What's missing is the infrastructure that keeps them alive between runs.

Today: Simulations

Digital twins run when someone launches a job and stop when it ends. Each run starts cold. Results sit on TACC until someone remembers to look. No continuous validation against live data. No alerting when models drift.

What this looks like in practice:
• Cole runs an MPACT simulation on TACC. Results land in an HDF5 file on Stockyard. Nobody is notified.
• Nick collects ZOC data daily. It goes to Box Cloud via nightly sync. There's no automated comparison against simulation predictions.
• Ondrej runs 10 MSR simulations/month. Each starts from scratch — no memory of previous runs, no parameter tracking across campaigns.
• Max hasn't collected any PiXie data yet. When he does, there's no pipeline to ingest it.
• When someone leaves, their simulation knowledge leaves with them — no institutional memory.

With NeutronOS: Live Twins

Twins become persistent and autonomous. They update when new measurements arrive. They alert when predictions diverge from reality. They accumulate fidelity over time. The Digital Twin Orchestrator manages their lifecycle without manual intervention.

What this looks like with NeutronOS:
• Model Corral: All model inputs (geometry, materials, BCs) are version-controlled and validated. The Orchestrator detects when inputs go stale.
• Simulation Tracking: Every TACC job is tracked — wall-clock, resources, convergence, results. Failures trigger Doctor Agent diagnostics.
• Continuous Validation: New ZOC/PiXie measurements flow into the lakehouse. The Calibration Agent compares predictions vs. reality and flags drift.
• Historical Analytics: Query any twin's prediction history. "Show me TRIGA power predictions vs. measured for the last 60 days." One command.
• Institutional Memory: Every simulation, every calibration, every parameter change is recorded. Knowledge stays when people leave.

The Twin Portfolio

NeutronOS manages the full lifecycle: Model Input Management (Model Corral) → Simulation Runtime & Tracking → Results Tracking & Publication → Twin Runtime Monitoring → Simulation Result Analytics (+ historical).
