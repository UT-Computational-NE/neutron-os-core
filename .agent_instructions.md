Agent Guidance — Repo-level instruction

Location: Neutron_OS root

Purpose
- Agents and automation should read this file first when operating on the repo. It documents the primary md→.docx converter and where generated artifacts must go.

Primary MD→DOCX converter
- Canonical script: `docs/_tools/md_to_docx.py` (production-ready). Use this first.
 - Canonical cost tool package (Python): `tools/cost_estimation` — preferred runtime home for the cost calculator.

Example usage (preferred):
- Convert single file:

  python docs/_tools/md_to_docx.py docs/analysis/phase1-cost-sizing/FORM-Cole-Physics.md docs/_tools/generated/FORM-Cole-Physics.docx

- Convert a folder (only changed files):

  python docs/_tools/md_to_docx.py docs/analysis/phase1-cost-sizing/ docs/_tools/generated/ --all --changed

Pandoc fallback
- If `md_to_docx.py` is unavailable, `pandoc` can be used as a simple fallback:

  pandoc docs/analysis/phase1-cost-sizing/FORM-Cole-Physics.md -o docs/_tools/generated/FORM-Cole-Physics.docx --toc --toc-depth=3

Output location and git behavior
- Output directory: `docs/_tools/generated/` (this repo ignores `docs/**/*.docx`, see `.gitignore`). Do not commit generated `.docx` files.
- If you need a specific docx to be tracked, add an exception in `.gitignore` and get approval.

Canonical Python packages and shims
- Canonical package location: `tools/cost_estimation`.
 - Package location: `tools/cost_estimation`.
 - Update existing imports to use `from tools.cost_estimation import ...` when importing from outside the `Neutron_OS` package.

Legacy / backup scripts
- Old or backup converters live under `docs/_archive/2026-01-22/` (e.g., `md_to_docx_backup.py`). Prefer the `docs/_tools/md_to_docx.py` script.

Agent responsibilities
- ALWAYS read this file first.
- Use `docs/_tools/md_to_docx.py` where possible and write outputs to `docs/_tools/generated/`.
- Respect `.gitignore` — do not add generated `.docx` to commits.

Contact
- If you find a failing converter or need a new feature, notify Ben and open an issue in the repo.
